Modules

The server uses a few modules that are imported into 'Submodules' to create a easier syntax for me to code with
while also keeping the code concise. However, these custom syntax can be confusing, so here is a general
overview of each one.

authhandler.py

the authhandler is responsible for generating, banishing, and verifying user AUTH keys.
This is a critical module

Imports:
String (builtin)
Random (builtin)
Jsonhandler as json (See jsonhandler.py)

Functions: 

generate_key:
Generates and logs an AUTH key

arguments: 
None

Returns:
key

Banish:
Removes keys

arguments:
key: The AUTH key to banish

Returns:
N/A

jsonhandler.py

The JSON handler is responsible for handling anything to do
with JSON. For the sake of convienience json.dumps and json.dump
is also stored here to not have to import json aswell


Imports:
json (builtin)

Functions : 

retrieve :
Returns the content of a JSON file in the form of a Python Dictionary

arguments : 
jsonfile: a string containing the path of the JSON file to retrieve

Returns :
contents of jsonfile as a dictionary

commit :
overwrites the jsonfile

arguments :
jsonfile: the jsonfile to commit to
dict: the dictionary to commit from

Returns :
N/A

isthere :
Checks if a value is in the provided dictionary

Arguments :
value: the value to search for
dict: the dictionary to search

Returns:
boolean: (True/False)

dump and dumps:
see https://docs.python.org/3/library/json.html

encryptor.py

The encryptor handles encryption/decryption


Imports:
python-gnupg (installed via requirements.txt)

Functions : 

Encrypt :
Encrypts the message using the users public key

arguments : 
content: the content to encrypt (string)
key: receipients public key

Returns :
final: the encrypted message
Decrypt :
Decrypts a message with the servers private key

arguments :
content: the encrypted message
key: the server public key
Returns :
final: the decrypted message


API

The server is designed for thrid party clients to be used instead of the primitive official CLI

Overview

Structure:

general user:
<serveraddress>
<serveraddress>/init 
<serveraddress>/messages
<serveraddress>/messages/send
<serveraddress>/banish

ADMIN:
<serveraddress>/admin
<serveraddress>/admin/init
<serveraddress>/admin/banish
<serveraddress>/admin/stop

Headers

AUTH: your auth key

Encrypted?: is the message encrypted

Password: the server password

PGP: : your public key

Content-type: the content type

AUTH:

The AUTH key is how the server knows who you are.
the only times you dont need the AUTH header is 
during a /init request

ENCRYPTED:

Encyrpted tells the server that your message has been
encrypted. This can be either 'True' or 'False' THIS
MUST BE A STRING!!! Not needed during a /, /init, or
a /banish request

PASSWORD:

The password header is only required during a / init request,
and only if the server is password protected. This is because
it is the most convenient for the user not having to input a 
password at every connection

PGP: :
the PGP: header is required when the ENCRYPTED header is, it is the users public PGP key

Content-type:
This is pretty self explanitory, required for POST and PUT requests

<serveraddress>

Request type: GET
Mandatory headers:
    -AUTH
    -PGP

Server address is used to get the general information  about the server

Response:
200 - successful
